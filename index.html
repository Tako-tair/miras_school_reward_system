How it should work

  1. The website loads student names from the Students sheet.
  2. The website loads action names and points from the Actions sheet.
  3. The website loads teacher login and class access from the Teachers sheet.
  4. When a teacher saves an action, the website sends it to Google Apps Script.
  5. Apps Script writes a new row to the Logs sheet.
  6. The website refreshes leaderboard, totals and filtered time-based views.

Access logic

Teacher sees only allowed classes

Teachers log in using email and password from the Teachers sheet.

Each teacher has a classes field. Example:

  • 5A,5B,5C means the teacher only sees these classes
  • ALL means the user is admin and sees all classes

This means there is no need to choose a teacher manually in the interface.

The logged-in teacher becomes the source of access rights for students, leaderboard and history.

Time filters can be used for today, this week, this month and all time analytics.
